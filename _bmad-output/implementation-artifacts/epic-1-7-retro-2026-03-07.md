## 二、未解决的 Review 项（按严重级别）

### CRITICAL

1. **`reject`/`other` Gate 命令为 stub 实现**（Story 3.2）
   - 位置: `src/commands/reject.ts`, `src/commands/other.ts`
   - 影响: 三态决策（approve/reject/other）是门控机制核心，目前仅 `approve` 完整实现
   - 风险: 门控流程无法执行拒绝和替代方向决策

### HIGH

2. **`handoff` 缺少审计事件和字段契约**（Epic 4）
   - 位置: `src/commands/handoff.ts:137`
   - 影响: 架构要求 `agent_intent -> agent_result/agent_compensation` 事件对，当前未实现
   - 风险: Agent 切换不可追溯，违反 adapter 两阶段协议

3. **`resume` preflight 缺少核心校验**（Epic 5）
   - 位置: `src/commands/resume.ts:10`
   - 影响: 缺少 checksum chain 验证、snapshot anchor 验证、schema/upcaster 兼容性检查
   - 风险: 恢复流程可能在数据不一致时继续执行，违反 hard-stop 约束

4. **`replay` 缺少 checkpoint 分段和错误处理**（Epic 5）
   - 位置: `src/commands/replay.ts:71`
   - 影响: 无 checkpoint 分段回放，静默吞掉损坏事件行
   - 风险: 确定性回放不可靠，违反 NFR8（deterministic replay）

5. **默认 Gate 策略不完整**（Story 1.2）
   - 影响: 架构要求 5 个默认策略（prd_freeze, architecture_freeze, high_complexity, release, fix_loop），当前仅部分实现

6. **Config 优先级解析未实现**（Story 1.3）
   - 影响: 架构要求 CLI flags > ENV vars > config.yaml 的优先级解析并标注来源，当前缺失

7. **事件日志写入和工作区文件锁未实现**（Story 1.4）
   - 影响: append-only 事件写入和 `.ai-dev/.lock` 文件锁是核心架构约束

### MEDIUM

8. **命令层直接读写 `.ai-dev/*`**（Story 1.4）
   - 影响: 违反架构分层（commands -> core -> 文件系统），增加耦合

9. **命令测试语义断言不足**（Story 1.1）
   - 影响: 仅检查退出码和 envelope 形状，存在 false positive 风险

10. **sprint-status.yaml 与 Story 文件状态不一致**（Story 1.2）
    - 影响: 多个 Story 文件标记 in-progress，但 sprint-status 标记 done

## 三、跨 Epic 模式分析

### 模式 1：文档质量急剧下降

- Epic 1 Story 1.1: 190+ 行，完整 dev notes, 文件清单, 多轮 review 记录
- Epic 2-3: 精简但可用
- Epic 4-5: 仅 epic-completed 汇总（17-33 行）
- Epic 6: **零文档**
- Epic 7: 仅 epic-completed 汇总

**影响:** 后期 Epic 缺乏实现决策记录，增加维护难度和知识断层风险。

### 模式 2：批量交付，速度换深度

- Epic 4/5/6 在同一 commit（`b4e18e5`）中完成
- Epic 7 紧随其后（`44f98c4`）
- 无独立 Story 文件，无逐步 review

**影响:** 功能覆盖面广但实现深度不足，多项核心架构约束被简化或跳过。

### 模式 3：测试可见性缺失

- 仅 Story 1.1 有测试记录（45 tests: 17 unit + 21 integration + 7 e2e）
- Epic 2-7 所有 Story 无测试信息
- 无测试覆盖率数据

**影响:** 无法评估代码质量和回归风险。

### 模式 4：架构规范执行不完整

以下核心架构约束未完全实现：

| 约束 | 状态 |
|------|------|
| Checksum chain（sha256 链式校验） | 未实现 |
| Append-only 事件写入通过 event-writer | 未确认 |
| Resume preflight hard-stop | 简化实现 |
| Adapter 两阶段协议（intent/result/compensation） | 未实现 |
| 工作区文件锁 | 未实现 |
| Config 优先级解析 | 未实现 |
| 命令层不直接访问文件系统 | 违反 |
| `--json` contract 测试 | 无记录 |

### 模式 5："Done" 标准不一致

- Story 1.1: 经历 2 轮 Codex review，13+ follow-ups，严格标准
- Story 1.2-1.4: 标记 done 但有 HIGH review 项未解决
- Epic 4-7: 无 review 记录直接标记 done

**影响:** "Done" 含义在项目生命周期中逐渐稀释。

### 模式 6：Epic 6 与 Epic 7 功能重叠

- Epic 6（运行时可观测性）和 Epic 7（决策日志与追溯）覆盖近乎相同的关注点：事件查询、过滤、导出、审计
- 无文档说明二者如何划分或是否共享代码
